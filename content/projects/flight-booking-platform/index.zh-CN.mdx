### 项目概述

主导关键任务机票预订平台的前后端整合系统现代化，服务 **每日 100 万+ 用户**。平台处理机票搜索、预订和支付流程，横跨美团 App 和微信小程序，需要高可靠性、正确性和低延迟。

### 主要成就

#### **性能优化**
- 使用 **延迟加载**、**预请求** 和 **请求合并** 优化 API 管道和请求策略
- 通过减少打包大小、tree-shaking 和资源预加载，将 **初始渲染速度提升 16.7%**
- 通过减少交互时间并改善用户体验，将 **预订转换率提升 3%**

#### **系统现代化**
- 将旧有模块现代化为采用模块化设计模式的 **可扩展架构**
- 通过代码重构和可重用组件开发，**长期维护成本降低 30%**
- 开发 **20+ 个可重用业务组件**（按钮、标签导航、显示卡片）以减少团队工作量

#### **跨平台一致性**
- 与平台部门协作，统一 **美团 App** 和 **微信小程序** 的用户体验
- 确保不同平台间的一致行为和数据流
- 在维持代码可重用性的同时，实现平台特定优化

### 技术挑战与解决方案

#### **首次内容绘制（FCP）优化**

**挑战：** 在高流量下持续减少预订流程的交互时间。

**实现方案：**

1. **打包大小减少**
   - 实现 **tree-shaking** 以消除未使用的代码
   - 使用 Babel 插件将 `import _ from 'lodash'` 转换为高效模块导入，如 `import throttle from 'lodash/throttle'`
   - 替换较大的第三方库（例如 **moment.js → day.js**）以最小化依赖大小
   - 使用图片压缩工具（TinyPNG）减少初始资源加载

2. **预加载策略**
   - 使用 **PN 文件** 机制进行动态资源加载与监听器
   - 提前发出请求并与主打包加载同步
   - 使用拦截器实现 **请求合并** 以避免冗余请求

3. **渲染优化**
   - 配置 **FlatList** 和 **VirtualizedList** 属性（`initialNumToRender`、`maxToRenderPerBatch`）以优化初始渲染
   - 预先计算单元格高度以改善列表渲染效率
   - 使用 **页面预加载** 和 **RAM 打包机制** 预加载下一页打包并重用短生命周期引擎实例
