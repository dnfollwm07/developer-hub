## 跨平台 AI 截圖系統 (Snippai)

**角色：** 前端工程師  
**期間：** 2025年5月 – 2025年8月  
**公司：** Snapbit LLC，美國懷俄明州謝里登  
**技術棧：** React, Electron, Node.js, TypeScript, REST API, OCR, LaTeX

---

### 專案概述

為 **Snippai**（跨平台 AI 截圖提取工具）建構全端桌面應用程式，支援 macOS 和 Windows。系統讓使用者能夠擷取截圖、使用 OCR 提取文字和表格，並將內容轉換為 Markdown、LaTeX 或 Excel 格式。

---

### 主要職責

#### **全端功能開發**
- 實作截圖擷取、OCR 處理和格式轉換的核心功能
- 使用 Electron 建構 macOS 和 Windows 的跨平台相容性
- 使用 React 設計直觀的桌面應用程式 UI/UX

#### **後端匯出管道**
- 開發 **Node.js 匯出管道**，支援 Markdown、LaTeX 和 Excel 格式
- 實作 **完善的錯誤處理**，具備全面的日誌記錄和狀態一致性保證
- 確保跨非同步操作和檔案 I/O 的資料完整性

#### **API 設計與優化**
- 設計並優化 **REST API** 以提升可靠性和可維護性
- 減少 macOS 和 Windows 流程間的跨平台不一致性
- 實作版本安全狀態同步以支援多平台

#### **CI/CD 與自動化**
- 自動化打包、簽名、發布管道和更新機制
- 透過標準化建置流程減少建置不一致性
- 使用自動化測試和部署管道加速發布週期

#### **協作與迭代**
- 與產品經理協作，釐清模糊需求並推動探索性研究
- 在維持高程式碼品質標準的前提下每週交付迭代
- 平衡功能開發速度與系統可靠性和可維護性

---

### 技術亮點

#### **跨平台架構**
- **Electron** 作為原生桌面應用程式框架
- **React** 用於 UI 元件開發
- **Node.js** 用於後端處理和檔案操作
- macOS 和 Windows 的平台特定優化

#### **匯出管道設計**
- 支援多種格式（Markdown、LaTeX、Excel）的模組化匯出系統
- 錯誤隔離和恢復機制
- 結構化日誌記錄用於除錯和監控
- 跨匯出操作的狀態一致性檢查

#### **建置與發布自動化**
- 兩個平台的自動化程式碼簽名
- 發布管道管理（穩定版、測試版、開發版）
- 增量更新系統以提供無縫使用者體驗
- 建置時完整性檢查以防止部署問題

---

### 影響

- 成功交付跨平台桌面應用程式
- 透過自動化減少建置不一致性
- 透過 CI/CD 整合加速發布週期
- 透過簡化工作流程提升開發者生產力
