## 跨平台 AI 截图系统 (Snippai)

**角色：** 前端工程师  
**期间：** 2025年5月 – 2025年8月  
**公司：** Snapbit LLC，美国怀俄明州谢里登  
**技术栈：** React, Electron, Node.js, TypeScript, REST API, OCR, LaTeX

---

### 项目概述

为 **Snippai**（跨平台 AI 截图提取工具）构建全栈桌面应用程序，支持 macOS 和 Windows。系统让用户能够捕获截图、使用 OCR 提取文本和表格，并将内容转换为 Markdown、LaTeX 或 Excel 格式。

---

### 主要职责

#### **全栈功能开发**
- 实现截图捕获、OCR 处理和格式转换的核心功能
- 使用 Electron 构建 macOS 和 Windows 的跨平台兼容性
- 使用 React 设计直观的桌面应用程序 UI/UX

#### **后端导出管道**
- 开发 **Node.js 导出管道**，支持 Markdown、LaTeX 和 Excel 格式
- 实现 **完善的错误处理**，具备全面的日志记录和状态一致性保证
- 确保跨异步操作和文件 I/O 的数据完整性

#### **API 设计与优化**
- 设计并优化 **REST API** 以提升可靠性和可维护性
- 减少 macOS 和 Windows 流程间的跨平台不一致性
- 实现版本安全状态同步以支持多平台

#### **CI/CD 与自动化**
- 自动化打包、签名、发布渠道和更新机制
- 通过标准化构建流程减少构建不一致性
- 使用自动化测试和部署管道加速发布周期

#### **协作与迭代**
- 与产品经理协作，厘清模糊需求并推动探索性研究
- 在维持高代码质量标准的前提下每周交付迭代
- 平衡功能开发速度与系统可靠性和可维护性

---

### 技术亮点

#### **跨平台架构**
- **Electron** 作为原生桌面应用程序框架
- **React** 用于 UI 组件开发
- **Node.js** 用于后端处理和文件操作
- macOS 和 Windows 的平台特定优化

#### **导出管道设计**
- 支持多种格式（Markdown、LaTeX、Excel）的模块化导出系统
- 错误隔离和恢复机制
- 结构化日志记录用于调试和监控
- 跨导出操作的状态一致性检查

#### **构建与发布自动化**
- 两个平台的自动化代码签名
- 发布渠道管理（稳定版、测试版、开发版）
- 增量更新系统以提供无缝用户体验
- 构建时完整性检查以防止部署问题
